Formydable

is a form validator library that is delightfully made in react. It is so light with unique setting and cognitive design. the rules are eloquent which will make the schema so powerful yet easy to understand, and implement on any kind of components. unlike any other form validator library, formydable is complex that allows you to bend and shape it on almost what ever you want. furthermore, formydable is capable on working both controllable and uncontrollable form fields and to know more just read the documentation below.

Installation #

npm i formydable
Imports #

import { useForm, useFormSupplier, FormSupplier, Validator, DoneTypingEvent } from "formydable"
Getting Started #

This is a react form validator that is delightfully made by mytabowrks which is blazingly fast and can be easily use and implement in any field components with ease.

import React from "react";
import { useForm } from "formydable";
const FormBasic = () => {
    const registerSchema = {
        fname: { label: "First Name", rules: "required|alpha|min:3" },
        lname: { label: "Last Name", rules: "required|alpha|min:3|max:20" }, 
    };
    const { formState, formUpdate } = useForm(registerSchema);
    const { fname, lname } = formState(); 
    
    return ( 
        <form> 
            <div className="n-form-field">
                <label>First Name</label>
                <input 
                    className="n-form-control"
                    name="fname" 
                    placeholder="first name..."
                    onChange={formUpdate}
                />
                {fname.isInvalid && <div className="error-feedback">{fname.message}</div>}
            </div>
            <div className="n-form-field">
                <label>Last Name</label>
                <input 
                    className="n-form-control"
                    name="lname"
                    placeholder="last name..."
                    onChange={formUpdate}
                /> 
                {lname.isInvalid && <div className="error-feedback">{lname.message}</div>} 
            </div>
        </form> 
    );
}
#render(<FormBasic />)
Tip

Rules validation order is from left to right. it will validate firstly the one on the left side which is the most priority. commonly required rule.

Customize Messages #

you can make your own static rule messages by using "failedIn" without affecting the rules messages model.

import React from "react";
import { useForm } from "formydable";
const CustomMesssages = () => {
    const registerSchema = {
        reaction: { label: "Reactions", rules: "required|alpha_space|min:10|max:20" }
    };
    const { formState, formUpdate } = useForm(registerSchema);
    const reaction = formState("reaction"); 
    
    return ( 
        <form>   
            <div className={`n-form-field${reaction.isInvalid

                    name="reaction"

                    reaction.failedIn === "min"

                    ? "That is too long"
                    : reaction.message }

Submitting Form #

form only exist to submit that is why it is not complete without submitting it on server side.

import React from "react";
import { useForm } from "formydable"; 
const FormWithSubmit = () => {
    const registerSchema = { 
        gender: { label: "Gender", rules: "required" },
        biography: { label: "Biography", rules: "required|alpha_space|min:20|max:100" },
        "languages[]": { label: "Language You Like", rules: "required|min:2|max:3" }
    };
    
    const { formState, formUpdate, formSubmit } = useForm(registerSchema);
    const { gender, biography, "languages[]": languages } = formState(); 
    const formSubmitHandler = formSubmit((event) => {
        if (event.isReady()) {
            MockRequest("/mock/url-encoded/request", {
                method: "POST",
                headers: {
                    "Content-Type": "application/x-www-form-urlencoded"
                },
                body: event.param()
            }) 
            .then((response) => { 
                Alert(response.json(), "success")
            })
            event.reset()
            
        } else {
            Alert("Failed! all field rules must fulfilled", "danger")
            event.locateFailed()
        }
    }); 
    const messageDisplay = (fieldstate) => fieldstate.isInvalid && <div className="error-feedback">{fieldstate.message}</div>
    
    return ( 
        <form onSubmit={formSubmitHandler}>  
            <div className="n-form-field">
                <label>Gender</label>
                <select className="n-form-control" name="gender" onChange={formUpdate}>
                    <option value="">Choose gender</option>
                    <option value="1">Male</option>
                    <option value="2">Female</option>
                </select>  
                {messageDisplay(gender)}
            </div> 
            <div className="n-form-field">
                <label>Biography</label>
                <textarea className="n-form-control" name="biography" onChange={formUpdate} />
                {messageDisplay(biography)} 
            </div>
            <div className="n-form-field">
                <label className="n-form-check">
                    <input className="n-form-control" type="checkbox" name="languages[]" value="js" onClick={formUpdate} />
                    js
                </label>
                <label className="n-form-check">
                    <input className="n-form-control" type="checkbox" name="languages[]" value="c#" onClick={formUpdate} />
                    c#
                </label>

                    <input className="n-form-control" type="checkbox" name="languages[]" value="c++" onClick={formUpdate} />

If you want to involve the unregistered field. You can always use FormEvent method formData that will return FormData instance.

Advance Design Pattern #

We use FormSupplier to provide formState, formUpdate and formRegistry to children components, and we use useFormSupplier to get FormSupplier provided to children components

import React from "react";
import { useForm, useFormSupplier, FormSupplier } from "formydable"; 
const FormWrapper = ({children, onSubmit}) => {
    const { formState, formUpdate, formSubmit, formRegistry } = useForm();
    
    const formSubmitHandler = formSubmit(onSubmit);  
    
    return ( 
        <FormSupplier value={{ formState, formUpdate, formRegistry }}>
            <form onSubmit={formSubmitHandler}> 
                {children}
                <button type="submit" className="n-btn">Submit</button>
            </form>
        </FormSupplier> 
    );
}
const InputWithRules = ({type = "text", name, label, rules, value = "", placeholder = "enter here...", id}) => { 
    const { formState, formUpdate, formRegistry } = useFormSupplier(); 
 
    formRegistry({ name, label, rules, value }) 
    
    const fieldstate = formState(name); 
    
    const validity = (
            fieldstate.isInvalid 
            ? "n-form-invalid" 
            : fieldstate.isValidated 
            ? "n-form-valid"
            : ""
        )
    return ( 
        <div className={`n-form-field ${validity}`}>
            <label htmlFor={id || name}>
                {label}
                {rules.includes("required") && (
                    <span className="label-required">*</span>
                )}
            </label>
            <input 
                className="n-form-control"
                id={id || name}
                type={type}
                name={name}
                value={type !== "file" ? fieldstate.value : undefined}
                placeholder={placeholder}
                onChange={formUpdate} 
            /> 
            <div className="n-invalid-feedback">
                {fieldstate.isInvalid && fieldstate.message}
            </div> 
            <div className="n-valid-feedback">
                <span role="img">✔️</span> Looks good!
            </div>  
        </div>
    );
}
const MockSubmit = (event) => {

    }
}
#render(
    <FormWrapper onSubmit={MockSubmit}>

    </FormWrapper>
)

Aliased Registry #

It is inevitable that some form fields are multiple with the same name on it, the problem is we wanted to add those new or other fields into form state registry. for that reason formydable use alias registry and make a counter measure about those stuffs.

import React from "react";
import { useFormSupplier } from "formydable"; 
const InputMultiple = (props) => {
    const [fields, setFields] = React.useState([
        {...props, alias: `unique-id-${Math.random()}`}
    ]); 
    return (
        <div className="n-form-field">
            <label htmlFor={props.name}>{props.label}</label>
            {fields.map((field, index) => {  
                const btnLabel = index === 0 ? "add" : "remove"
                const handleClick = () => {  
                    setFields((state) => {
                        return index === 0 
                        ? [
                            ...state,
                            {
                                ...props,
                                alias: `unique-id-${Math.random()}`,
                            }
                        ] 
                        : state.filter((value) => value.alias !== field.alias)
                    }) 
                } 
                return ( 
                    <InputWithAliasRules key={field.alias} btnLabel={btnLabel} onClick={handleClick} {...field} /> 
                )   
            })} 
        </div>
    )
}
const InputWithAliasRules = ({  type = "text", name, alias, label, rules, placeholder = "enter here...", btnLabel, onClick}) => { 
    
    const { formState, formUpdate, formRegistry } = useFormSupplier(); 
 
    formRegistry({ name, label, rules, alias })
    
    const fieldstate = formState(alias);
    
    const handleFieldChange = (event) => { 
        formUpdate({ target: event.target }, alias);
    }; 
    
    const validity = (
        fieldstate.isInvalid 
        ? "n-form-invalid" 
        : fieldstate.isValidated 
        ? "n-form-valid"
        : ""
    )
    return (   
        <div className={`n-form-field ${validity}`}>
            <div className="n-input-group"> 
                <input 
                    className="n-form-control"
                    id={name}
                    type={type}
                    name={name}
                    data-alias={alias}
                    placeholder={placeholder}
                    onChange={handleFieldChange} 
                /> 
                <div className="n-input-group-append">
                    <button
                        className="n-btn" 
                        type="button"
                        onClick={onClick}>
                        {btnLabel}
                    </button>
                </div>  
            </div>
            <div className="n-invalid-feedback">
                {fieldstate.isInvalid && fieldstate.message}
            </div> 
            <div className="n-valid-feedback">
                <span role="img">✔️</span> Looks good!
            </div>
        </div>   
    );
}
#render(
    <FormWrapper onSubmit={MockSubmit}>
        <InputMultiple name="favorites" label="Favorites" rules="required|alpha|min:2" />
    </FormWrapper>
);

FormUpdate With DoneTypingEvent #

It is use to fire the event after user is done typing, that will save a lot of unessesary execution of state changes while typing.

import React from "react";
import { useFormSupplier, DoneTypingEvent } from "formydable"; 
const InputDoneTypingWithRules = ({  type = "text", name, label, rules, placeholder = "enter here...", id}) => { 
    const { formState, formUpdate, formRegistry } = useFormSupplier(); 
 
    formRegistry({ name, label, rules }) 
    
    const fieldstate = formState(name);
    
    const handleDoneTyping = DoneTypingEvent((event) => {
        formUpdate({ target: event.target });
    }, 300); 
    
    const validity = (
        fieldstate.isInvalid 
        ? "n-form-invalid" 
        : fieldstate.isValidated 
        ? "n-form-valid"
        : ""
    )
    return ( 
        <div className={`n-form-field ${validity}`}>
            <label htmlFor={id || name}>
                {label}
                {rules.includes('required') && (
                    <span className='label-required'>*</span>
                )}
            </label>
            <input 
                className="n-form-control"
                id={id || name}
                type={type}
                name={name}
                placeholder={placeholder}  
                {...handleDoneTyping}
            /> 
            <div className="n-invalid-feedback">
                {fieldstate.isInvalid && fieldstate.message}
            </div> 
            <div className="n-valid-feedback">
                Looks good!
            </div>  
        </div>
    );
}
#render(
    <FormWrapper onSubmit={MockSubmit}>   
        <InputDoneTypingWithRules name="done-typing" label="With DoneTypingEvent" rules="required|alpha|min:20|max:25" />
        <InputWithRules name="normal-change-event" label="Without DoneTypingEvent" rules="required|alpha|min:20|max:25" /> 
    </FormWrapper>
)

const strong_password = {
